# Data and reproducibility

Most results are synthetic and generated by the scripts in `scripts/`, each fixing its own seed
via `numpy.random.default_rng`. External-data studies (CRSP via WRDS, CMS dimuon, SDSS DR18,
UNSW-NB15) are documented below.

## v2 correction and sweep (read first)
Every ISJ/Botev number through v1 was produced by the KDEpy `improved_sheather_jones`, which
undersmooths ~2.6x versus the Botev-Grotowski-Kroese fixed point (h ~ 0.075 vs ~0.198 on N(0,1)
at n=5000; AMISE optimum 0.193). v2 embeds the direct fixed point (`scripts/ad_kde_v31.py`;
KDEpy dropped; `scripts/ad_kde_v30.py` is a compatibility shim) and regenerates every ISJ
result on identical draws. The Marron-Wand benchmark is now a seven-estimator sweep over
n in {50,100,200,500,1000,2000,5000}, 50 reps per cell, run by `scripts/exp_benchmark_v28.py`
(draws: default_rng(1000*r+n), density-independent; chunked runs write
results/exp_benchmark_v28_n{N}[_chunk..].json; `--merge` produces exp_benchmark_v28_sweep.json).
`scripts/make_ranksweep_v2.py` assembles fig_ranksweep.pdf and the tab:ranksweep rows from the
results; ranks use average-tie ranking of reported-precision per-density means, so figure and
tables are mutually recomputable. The anchor tables (n=100/500/5000) carry the same-draw
printed columns of the v1 tables with the ISJ column regenerated. The GEMMULE column of v1 is
removed (7 estimators). The UNSW cyber table's ISJ column awaits a local rerun with
`scripts/exp_cyber_unsw_v28.py`.

## Kernel density estimation (paper Secs. III-VI)
| Script | Seed | Figures | Parameters |
|---|---|---|---|
| `ad_kde_v31.py` | 2024 (benchmark); 7, 11, 5 (figures) | `fig_kde_examples.pdf`, `fig_kde_hard.pdf`, `fig_kde_heaped.pdf` | Marron-Wand densities; n in {200, 2000}; R = 60/40 reps; heaped experiment R = 40; eval grid 1600 |

Densities: Gaussian, Bimodal, Kurtotic, Claw, Asymmetric claw, Smooth comb, Discrete comb,
Strongly skewed. Methods: best fixed bandwidth (ISE-minimizing single h), Silverman's rule,
the diffusion estimator at the improved Sheather-Jones bandwidth (Botev), the ECF plug-in
selector (Chiu), the adaptive square-root-law variable-bandwidth estimator (Abramson), the
AD bandwidth selector (AD-bw), and the AD-Wiener adaptive estimator. The AD methods support
two floor-stripping modes via `--strip {simple,residue}`: `simple` (default) uses the known
1/n floor; `residue` uses the algebraic-residue floor `median(|ECF|^2)/ln 2` with the soft
Wiener gain. Table I (exact samples, default strip) and Table II (heaped data, simple vs
residue) are in units of 1e-3 and are reproduced by `python3 scripts/ad_kde_v31.py`. The
benchmark takes roughly two to four minutes.

Heaped experiment: each sample is rounded to one decimal (0.1 grid) before estimation; this
aliases mass to high frequencies and lifts the true floor above 1/n, so the fixed-floor
strip fails (catastrophically for the Wiener estimator) while the residue floor adapts.

`KDEpy` is required only for the Botev/ISJ baseline (`pip install KDEpy`). If absent, the
script still runs and that column falls back to Silverman; the build does not fail.

## Enabling estimator: signal bandwidth (paper Sec. VII)
| Script | Seed | Figures | Parameters |
|---|---|---|---|
| `ad_bw_core_v1.py`    | 7  | `fig_mechanism.pdf`, `fig_bw_sweep.pdf` | M=512, fs=512 Hz, K=64 bins, T=400; mechanism check at M=64 |
| `ad_bw_compare_v1.py` | 11 | `fig_b99_compare.pdf` | M=512, fs=512 Hz, K=64, T=400; B99 = 99% contiguous occupied bandwidth |
| `ad_bw_chirp_v1.py`   | 19 | `fig_chirp.pdf` | M=512, fs=512 Hz, LFM sweep 80->220 Hz, 20 dB; chirp-rate grid 601 points |

Bin spacing Delta f = fs/M = 1 Hz, so K=64 bins equals 64 Hz. Single-observation (random
band, stripped, high SNR) gives B_eff ~ 33 Hz, matching (K+1)/2 = 32.5; multi-snapshot
recovery L=1 ~33, L=8 ~57, L=32 ~62 Hz; chirp raw 137.8 Hz, reconstructed swept 140.5 Hz,
instantaneous 1.1 Hz (true swept 140 Hz).

## Reproduce
```
./build.sh --check
```
regenerates all six figures, recompiles the PDF, and verifies every artifact. A run that
does not reproduce the reported numbers is a defect.

Diagnostic + symmetrizing-warp experiment (ad_kde_v31.py): coherent_effdim reports D2 of the
stripped squared-ECF spectrum; ad_warp uses scipy.stats.yeojohnson to symmetrize, AD-Wiener in
the warped domain, and the analytic Yeo-Johnson Jacobian to map back. Seed 2024, 40 draws,
n=2000, same generator state as the heaped experiment.

Dilation-group wavelet benchmark (ad_kde_v31.py): ad_wavelet is a translation-invariant
(12-shift cycle-spun) sym6 soft-threshold estimator of the binned density; the noise floor is
the std of the finest two detail levels times sqrt(2 log M). Requires pywavelets (pywt); if
absent the function falls back to AD-Wiener. Exponential-jump case uses range [-1,9], M=2048,
seed 2024, 40 draws, downstream of the warp experiment in the same generator stream.

## v9 experiments (new)

### Multivariate symmetry concentration (`exp_multivariate_v9.py`)
- Dimension d=32; true covariance circulant from an AR(1) power spectrum (rho=0.6) plus 1e-3 I;
  true effective dimension D_2 ~ 15.1.
- Snapshots n in {8,16,24,32,48,64,128,256}; reps=60; seed=2024; held-out set 4000 draws.
- Metric: excess held-out Gaussian negative log-likelihood over the oracle; effective dimension.
- Plugin: `match_group` (default cyclic) then `reynolds`. Reproducible without the reference engine.

### Leptokurtic tail risk (`exp_financial_v9.py`)
- Returns ~ 0.9 N(0, 0.008^2) + 0.1 N(0, 0.030^2); excess kurtosis 8.7.
- True VaR_1% = -0.0384, VaR_5% = -0.0160, ES_1% = -0.0526.
- n=1000 returns; reps=200; seed=2024; grid [-0.2, 0.2] with 20001 points.
- Methods: Gaussian fit, KDE (Silverman), AD-Wiener (simple strip), Mixture (`fit_mixture` plugin,
  default Gaussian-mixture EM + BIC, kmax=4). Reproducible with the bundled defaults alone.

### Plugin dependencies

  (full bundle only, wired by `reference/wire_reference.py`), not in the `github/` image. An

### SDSS galaxy-redshift spectrum (paper Sec. Real-Data Validation on Galaxy-Redshift Spectra)
- `exp_sdss_redshift_v16.py`: reads a SkyServer CSV export of SDSS DR18 SpecObj galaxy redshifts,
  column z. The CSV begins with a `#Table1` comment line, so it is read with `comment='#'`.
  Obtain it from the SkyServer DR18 SQL page (https://skyserver.sdss.org/dr18/SearchTools/sql,
  Output Format = CSV, press Submit) with:
      SELECT s.z FROM SpecObj AS s
      JOIN dbo.fGetNearbyObjEq(200.0, 0.0, 300) AS n ON n.objID = s.bestObjID
      WHERE s.class='GALAXY' AND s.zWarning=0 AND s.z BETWEEN 0.01 AND 0.25
  (the five-degree-radius cone about (ra,dec)=(200,0); fGetNearbyObjEq can be slow at this radius,
  so the coordinate box ra in [197.5,202.5], dec in [-2.5,2.5] with the JOIN removed is a faster
  near-equivalent). Then `python3 scripts/exp_sdss_redshift_v16.py --csv sdss_z.csv`. No seed beyond
  the fixed 70/30 split (seed 0): the data are fixed.
- 6,684 galaxies in [0.01,0.22]. Held-out mean NLL (lower better): naive KDE -1.876, AD-bw -2.011,
  AD-Wiener -2.009. Both AD estimators improve by ~0.13 nat; the gain is the spectral cutoff (the
  selector and Wiener estimator are indistinguishable). Stability: across 200 random 70/30 splits
  the AD-Wiener advantage over naive is +0.133 nat (sd 0.013), 100% win-rate. The advantage requires
  the well-sampled cone; a ~500-galaxy pencil beam does not separate it from split variation
  (+0.09 nat, sd 0.23, ~70% win-rate). Writes fig_sdss_redshift.pdf (full n(z) + dominant-wall zoom).
- SDSS data are public; the exported CSV is not redistributed in the bundle.

### Synthetic-data fidelity generator (paper Sec. A Synthetic-Data Application)
- `exp_datagen_v30.py`: fully synthetic, no external data. Inverse-CDF exact sampler from any target
  pdf grid; Huber eps-contamination G_eps=(1-eps)f+eps c (eps = literal error knob); optional
  Gaussian jitter. Target library includes 'halfhalf' (smooth bimodal left half, Marron-Wand claw
  right half), 'bimodal', 'kurtotic'. N bounded to [2^9,2^18]. Default run reproduces the halfhalf
  tie-case: D_2=2.5; per-half ISE (x1e3, N=8000, seed 0, exact eps=0): naive 2.07/24.13/26.20,
  mixture(GMM kmax=6) 0.09/20.14/20.23, AD-Wiener 0.43/0.75/1.18. Writes fig_datagen.pdf.
  Run: `python3 scripts/exp_datagen_v30.py`. The mixture backend is the default GMM-EM+BIC plugin
  (an external mixture library may be registered through the same hook).
